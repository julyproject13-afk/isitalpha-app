# 📊 ЗАДАЧА (сессии/Девину) — телеметрия воронки isitalpha: собрать, проверить, показать где рвётся

> Самодостаточно. research/analytics, без денег/ключей/торговли. Коммит локально.

## Контекст (что уже есть в коде)
`scripts/app.py` пишет **анонимную пофазную воронку** в `funnel.json` (без PII), с разбивкой по `utm_source`. Шаги (порядок = путь клиента):
```
FUNNEL_STEPS = ("land", "validate_click", "validate_run", "verdict_shown", "paywall_view", ... , sale)
```
- `land` — зашёл на /validate (сервер, стр. 452).
- `validate_click`, `verdict_shown` — клиентские beacon через `/api/track` (стр. 473).
- `validate_run` — сервер реально прогнал валидацию (стр. 513).
- `paywall_view` — показ пейвола (стр. 517).
- Продажа — в `metrics.json` (attempts/sales/revenue).
- Отчёт: endpoint **`/api/report`** (стр. 393) отдаёт `{report, metrics, funnel}`, защищён `REPORT_KEY` (env).

## ⚠️ Ключевой момент — данные ЖИВЫЕ на СЕРВЕРЕ, не локально
Локальный `funnel.json` = dev-мусор (`land:1, validate_run:2`). Реальные числа — в `funnel.json` НА СЕРВЕРЕ (/opt/isitalpha) + `/api/report` продакшена. Их достаёт владелец (VNC/ключ) — см. раздел «Вход владельца».

## Что сделать
1. **Проверить качество телеметрии (аудит):**
   - Все ли шаги реально инкрементятся? (симулировать путь: land→validate_run→verdict_shown→paywall_view, проверить, что funnel.json растёт корректно, по источнику и глобально).
   - Нет ли дыр: боты/healthcheck не считаются? (в коде есть фильтр `is_probe` — проверить). Клиентские beacon (`/api/track`) доходят?
   - Порядок шагов монотонный (нельзя paywall_view без validate_run)?
2. **Собрать сводку воронки** из живого `funnel.json` (даст владелец): по каждому шагу — сколько дошло, **% отвала между шагами**, разбивка по utm_source. Формат: таблица «шаг → N → conversion% → drop%».
3. **Найти, ГДЕ рвётся:** самый большой отвал между какими шагами. Гипотезы почему (напр. land→validate_run = не вводят данные; verdict_shown→paywall_view = вердикт получили, платить не идут; paywall_view→sale = ценник/оплата).
4. **Мини-дашборд** (опц.): простая страница/скрипт, читающий funnel.json → таблица + % отвала, чтобы владелец видел вживую.

## Вход владельца (его рука — данные с сервера)
- Через VNC на сервере: `cat /opt/isitalpha/funnel.json` и `cat /opt/isitalpha/metrics.json` → прислать содержимое.
- Или `curl "http://localhost:PORT/api/report?key=$REPORT_KEY"` на сервере.

## Приёмка
- Аудит телеметрии: PASS/список дыр.
- Сводка воронки с %-отвалами по шагам и источникам.

- Честно: если живых данных мало (трафика почти нет) — так и сказать, не выдумывать проценты.
