## 2. МЕТОДОЛОГИЯ 🟡

### 2.1 Фундаментальные границы return-only анализа (честно признаны)
Движок **не может** определить:
- **Скрытую бету** — переодетый лонг рынка (гладко в бычьем → крах в рецессию)
- **Нереализованный хвост** — шорт-вол стратегия (гладкий ряд до краха, которого ещё не было в данных)
- **Ёмкость** — может ли стратегия принять реальный капитал
- **Покрытие кризиса** — были ли 2008/2020/2022 в данных

### 2.2 Честен ли продукт про свои границы?
**Частично.** В `legal.html` (disclaimer) и в `note` вердикта сказано: «survivorship caveats apply», «not investment advice». Но **на самом экране валидации** (`validate.html`) границы не акцентированы. Пользователь видит «REAL ✅» и может решить, что стратегия безопасна для денег.

**Рекомендация (средний приоритет):**
- В вердикт REAL добавить 1-строчную метку: *«This checks historical robustness only — not hidden beta, capacity, or crisis coverage.»*
- Поля Биржа/Актив (необязательные) → бенчмарк для беты + правильные издержки. Уже запланировано.
- Режим-разрез (первая/вторая половина ряда) — cheap check, стоит добавить.

### 2.3 Deflated threshold — корректен
`_deflated_threshold` использует Bonferroni: `NormalDist().inv_cdf(1 - α/(2·n_trials))`. При n_trials=10 порог ≈2.58, при 100 → ≈3.29. Математически верно. Консервативнее, чем Holm-Bonferroni (строже = честнее для бренда).

### 2.4 Аннуализация
`periods_per_year` пробрасывается корректно через весь пайплайн (validate → fold_sharpes → _annual_sharpe). День/неделя/месяц — работает (стресс-тесты 17-18 подтверждают).

## 5. БЕЗОПАСНОСТЬ 🟠

### 5.1 IPN-подпись (NOWPayments) ✅
`np_ipn_valid` проверяет HMAC-SHA512 отсортированного JSON, пробует оба варианта (с/без escaped slashes для совместимости с PHP json_encode). Использует `hmac.compare_digest` (timing-safe). **Корректно.**

### 5.2 Обход пейволла ⚠️
- **⚠️ ДЕФЕКТ #2 (СРЕДНИЙ): `gate()` проверяет `is_paid(order)`, но order берётся из тела POST-запроса клиента.** Если злоумышленник знает (или перебирает) order_id оплаченного чужого заказа, он может подставить его и получить полный отчёт бесплатно.
- Order_id = `"isa_" + secrets.token_hex(8)` = 16 hex символов = 2^64 вариантов. Перебор нереалистичен. **Риск: LOW на практике**, но архитектурно order привязан к оплате, а не к конкретным данным → один order_id = бессрочный unlock для любых данных. Рекомендация: привязать order к хешу ряда или к IP/сессии (после масштабирования).

### 5.3 Инъекции
- `body.get("returns", "")` → `parse_returns_csv` → только `float()` конверсия. Безопасно — нет eval/exec.
- `n_trials` → `int()`. Безопасно.
- Файловый путь: `os.path.normpath(p).lstrip("/")` для статики. **Но:** `os.path.normpath` на Unix нормализует `..`, а `lstrip("/")` убирает leading slash. Результат: `../../etc/passwd` → `etc/passwd` (после normpath) → `lstrip("/")` → `etc/passwd` → `os.path.join(WEB, "etc/passwd")` → не существует → 404. **Безопасно** (не выходит за WEB).
- `/learn/` slug: `"".join(c for c in ... if c.isalnum() or c == "-")` — жёстко фильтрует. Безопасно.

### 5.4 DoS
- Нет rate-limiting. Один клиент может слать тысячи POST /api/validate. Каждый вызов validate() — O(n) по длине ряда. Ряд в 100K точек — заметная нагрузка.
- **⚠️ ДЕФЕКТ #3 (СРЕДНИЙ): нет ограничения на размер входных данных.** `self.rfile.read(n)` читает Content-Length без лимита. Рекомендация: ограничить body ≤1MB и ряд ≤10K точек.

### 5.5 Приватность
- **Ряды не логируются и не хранятся.** `log_message` = `pass`. validate() — pure function, результат возвращается и забывается. **Соответствует legal.html.**
- metrics.json хранит только агрегаты (attempts, sales, revenue). Order_id — без привязки к данным.
- paid_orders.txt — хранит только order_id (hex-строки), без данных пользователя.
- **Email capture на landing → localStorage only** — не утекает на сервер. Но см. замечание по UX (обман ожиданий).

### 5.6 Секреты
- `NP_API_KEY`, `NP_IPN_SECRET`, `TG_TOKEN` — из bootstrap.env (в .gitignore). Не хардкожены. ✅
- `REPORT_KEY` — простой ключ для /api/report (бизнес-сводка). Передаётся в query string (GET) → может попасть в логи nginx. Рекомендация: перенести в header.

**3 конкретных хода поднять конверсию:**
1. **Пакет / кредиты** — «5 проверок за $29» (вместо $45) — стимул к повторным + привлечение «строителей стратегий»
